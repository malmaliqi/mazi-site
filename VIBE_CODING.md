## 🤖 Instructions For The AI Agent

> **READ THIS CAREFULLY BEFORE DOING ANYTHING.** The person you're working with is completely new to all of this — they've never coded before and they're learning by building real projects. Here's how you should operate throughout the ENTIRE project:
>
> - **Explain everything like they're 15.** No jargon without a simple one-liner explanation right after. If you say "API," immediately explain what that means in plain english. If you set up a "server," tell them what a server actually is and why they need one. If you use a "library," explain what that word means in coding. EVERY. SINGLE. TIME.
> - **Teach as you build.** After completing each major step, give a quick casual **"🧠 What just happened?"** summary. 2-3 sentences max. Keep it light and conversational — no textbook energy. Think "cool older brother explaining things" not "professor giving a lecture."
> - **Name the technologies you're using and WHY.** When you pick a tool, framework, or library — say what it is, what it does, and why you're using it instead of something else. Example: *"We're using Next.js — it's a popular framework for building websites with React. I'm picking it because it makes routing (moving between pages) easy and we can deploy it for free."*
> - **Don't dump giant walls of code without context.** Walk them through what's happening in each file, even if briefly. Point out the parts that matter.
> - **Celebrate small wins.** When something works for the first time — the page loads, a button works, data shows up — acknowledge it. Hype them up. Momentum is everything.
> - **If something breaks, explain WHY it broke** in simple terms before fixing it. Errors are learning moments, not failures.
> - **Suggest what to try or tweak** once each milestone is done — help them feel creative ownership. "Try changing this color," "What if we added a section here," "Want to try a different layout?"
> - **Be patient.** If they ask a "dumb" question, answer it fully and kindly. There are no dumb questions when you're learning.
> - **Keep the energy up.** This person is building real things from zero. That's huge. Remind them of that.
> - **Be proactive.** Don't wait for detailed instructions. When they say "let's build Project X," YOU take the lead — read the brief, ask them a few quick questions about their preferences (vibe, colors, what content they have ready), and then start building step by step. Guide them through the whole process. Think of yourself as a co-builder who happens to know how to code, not a tool waiting for perfect instructions.
> - **Ask them questions along the way.** "Do you like this layout or should I try something different?" "What color scheme fits your brand?" "Do you have photos ready or should we use placeholders for now?" Make it collaborative.

### Project 1: Your Music Promo Site
**Tech:** HTML + CSS (the raw basics, no frameworks)
**Difficulty:** 🟢 Baby steps
**What you'll learn:** How websites are actually built from scratch, how HTML structures content, how CSS makes it look good, how to embed media
**Time estimate:** 1-2 hours

**The brief:**
Build a promo website for yourself as an artist. This is YOUR site — the place you send people when they ask "where can I find your stuff?" Include:
- A hero section with your artist name, a tagline or one-liner, and a hard-hitting photo or visual
- An "About" section — your story, your sound, where you're from
- A music/video section — embed YouTube videos, SoundCloud tracks, Spotify links, whatever you've got
- A photo gallery — press shots, studio shots, performance pics
- Social links — Instagram, TikTok, YouTube, whatever platforms you're on
- A contact section — for bookings, collabs, press

Make it match your brand. Dark and moody? Neon and loud? Minimal and clean? Your call. This is your identity.

**Just tell the agent:**
> *"Let's build Project 1 from VIBE-CODING.md"*

**Stretch goal:** Make it responsive so it looks good on phones too.

### Project 2: Mom's Painting Portfolio
**Tech:** HTML + CSS + JavaScript
**Difficulty:** 🟢🟡 Leveling up
**What you'll learn:** JavaScript basics, making things interactive (click events, showing/hiding things), working with images, building a second site from scratch
**Time estimate:** 2-3 hours

**The brief:**
Your mom is a painter. She deserves a beautiful site to showcase her art. Build her a portfolio/gallery site:
- A hero section with her name and a tagline (e.g. "Original paintings by [Name]")
- An "About the Artist" section — her story, her style, her inspiration
- A painting gallery — a clean grid of her work. When you click a painting, it opens up big in a lightbox/modal with the title, dimensions, medium, and a short description
- Organize paintings by collection or category if she has different styles/series
- A contact section — for inquiries, commissions, purchases

Make it elegant and artsy. Think high-end gallery, not a cluttered Etsy page. Let the art breathe.

**Just tell the agent:**
> *"Let's build Project 2 from VIBE-CODING.md"*

**Stretch goal:** Add a "Featured" or "New Work" section at the top that highlights her latest pieces.

### Project 3: Kosovo Job Market Analysis
**Tech:** Python (data analysis & visualization)
**Difficulty:** 🟡 New territory
**What you'll learn:** Python basics, how to read and process data, how to make charts and visualizations, how to extract insights from raw data — a completely different skill from building websites
**Time estimate:** 2-4 hours

**The brief:**
You'll be given a CSV file with scraped job listing data from KosovaJob (a friend will send it to you). Your job is to analyze it and pull out interesting insights about the Kosovo job market. Things like:
- What industries are hiring the most?
- What are the most in-demand skills?
- Which cities have the most listings?
- What's the distribution of job types (full-time vs part-time vs remote)?
- Any trends in salary ranges?
- What experience levels are most companies looking for?

Create clean visualizations — bar charts, pie charts, heatmaps, whatever tells the story best. The goal is to turn a messy spreadsheet into clear, visual insights that anyone could understand.

**Just tell the agent:**
> *"Let's build Project 3 from VIBE-CODING.md"*

**Stretch goal:** Generate a full "Kosovo Job Market Report 2025" with all your charts and insights, exported as a PDF or HTML page you can share.

### Project 4: Your Music Site — Rebuilt in Next.js
**Tech:** Next.js (React framework), Tailwind CSS
**Difficulty:** 🟡🟠 Framework time
**What you'll learn:** Why frameworks exist, what React/components are, routing (multiple pages), how modern web apps are built vs plain HTML, deploying to the real internet
**Time estimate:** 3-4 hours

**The brief:**
Remember Project 1? That HTML + CSS site you built? Now rebuild it — but with Next.js. Same concept, same content, but this time with a real framework. Why? Because:
- You'll learn what React components are (reusable building blocks)
- You'll learn routing (clean URLs for each page like `/about`, `/music`, `/contact`)
- You'll be able to deploy it live on Vercel for free with a real URL
- You can add features that would be painful in plain HTML (a music player, animations, dynamic content)

Take everything you learned from Project 1 and make it BETTER. New layout, new ideas, new features you wish you had the first time.

**Just tell the agent:**
> *"Let's build Project 4 from VIBE-CODING.md"*

**Stretch goal:** Deploy it live on Vercel. Get a custom domain if you're feeling bold. Send the link to people. This is your real site now.

Time to turn mom's portfolio into a STORE. People shouldn't just look at the paintings — they should be able to buy them. Build:

- Checkout with Stripe (real payment processing — you can use Stripe's test mode so no real money moves while you're building)
- An admin page or simple way for her to add new paintings, update prices, mark things as sold
- Categories/collections to browse by style or series

This is a real business tool. When this is done, she can actually sell paintings online.

**Just tell the agent:**

### Project 6: KosovaJob Scraper + AI Analysis
**Tech:** Python, web scraping (BeautifulSoup/Scrapy), AI API (Claude or OpenAI)
**Difficulty:** 🔴 Full circle
**What you'll learn:** Web scraping (getting data from websites automatically), working with APIs (talking to external services), how AI APIs work, putting multiple technologies together
**Time estimate:** 4-6 hours

**The brief:**
In Project 3, a friend gave you the data. Now YOU get the data yourself. Build a Python tool that:

**Part 1 — The Scraper:**
- Goes to KosovaJob and scrapes current job listings automatically
- Pulls out key info: job title, company, location, description, requirements, date posted
- Saves everything in a clean structured format (CSV or JSON)

**Part 2 — AI-Powered Analysis:**
- Sends the scraped data to an AI API (Claude or OpenAI)
- The AI enriches each listing — extracting required skills, experience level, estimated salary range, industry category, whether it's remote-friendly, etc.
- Generates a summary report: market trends, top skills in demand, most active companies, insights that aren't obvious from the raw data
- Compare the AI's analysis with what you found manually in Project 3 — what did the AI catch that you missed?

This project ties together everything: getting data (scraping), processing it (Python), and using AI to make it smarter (APIs). You're building a real automation pipeline.

**Just tell the agent:**
> *"Let's build Project 6 from VIBE-CODING.md"*

**Stretch goal:** Set it up to run automatically on a schedule (daily/weekly) and email you a fresh report.
